# COSMO 앱 Product Requirement Document (PRD) v1.0

> 작성일: 2026년 5월  
> 대상 버전: MVP v1.0  
> 플랫폼: iOS / Android / Web

---

## 1. 앱 컨셉 정의

### 앱명: COSMO (코스모)
**부제:** 오늘 밤 하늘이 궁금해지는 우주 탐험 앱

### 핵심 컨셉
> "천문학을 가르치는 앱이 아니라, 천문학을 사랑하게 만드는 앱"

### 포지셔닝
```
초등학생용 우주판 마인크래프트 + 포켓몬 도감 + 듀오링고 + 우주 유튜브 쇼츠
```

---

## 2. 핵심 사용자 분석

### 주요 타겟
- **연령**: 초등학교 4~6학년 (만 10~12세)
- **특성**: 마인크래프트 열성 팬, 탐험·수집·제작·건설 요소 선호
- **사용 맥락**: 하교 후(15~17시), 저녁 자유시간(19~21시)
- **1회 목표 사용 시간**: 3~15분

### 페르소나
| 페르소나 | 특징 | 앱에서 원하는 것 |
|---|---|---|
| 이준혁 (11세) | 마인크래프트, 수집 욕구 강함 | 재미 + 친구에게 자랑 |
| 박서연 (10세) | 스티커 수집, 예쁜 것 좋아함 | 예쁜 비주얼 + 성취감 |
| 최민준 (12세) | 블랙홀·외계인 관심, 유튜브 시청 | 심층 콘텐츠 + 지적 자극 |

## 3. 설계 원칙

### 원칙 1. 교육보다 호기심 우선
- ❌ "목성은 태양계에서 가장 큰 행성입니다"
- ✅ "만약 지구가 농구공이라면 목성은 자동차 크기일까?"

### 원칙 2. 텍스트보다 시각적 경험
- 카드형 콘텐츠, 애니메이션 중심, 인터랙션 우선

### 원칙 3. 학습보다 탐험
- 앱 구조를 "교과서"가 아니라 "우주 탐험"처럼 설계

### 원칙 4. 우주를 탐험하는 블록 월드처럼 느껴야 함
활용 요소: 탐험, 채굴, 제작, 수집, 건축, 업그레이드, 생존, 발견, 보물찾기, 업적

### 원칙 5. 천문학은 퀘스트 속에 숨겨져야 함
- 화성 광물을 채굴하다가 → 화성이 왜 빨간지 알게 됨
- 토성 고리 탐사 퀘스트 → 토성의 구조 이해
- 블랙홀 연구 미션 → 중력 개념 학습

## 4. 전체 기능 목록

### Core Features (MVP)
- ✅ 퀘스트 보드 홈 (MC 캐릭터 퀘스트 기버)
- ✅ 바이옴 탐험 (10개 행성, 위험도, 자원 채굴)
- ✅ 제작소 (15종 자원, 8종 레시피)
- ✅ 우주 기지 (Lv.1~5, 업그레이드, 업적)
- ✅ AI 동반자 (6개 MC 캐릭터, Claude API)
- ✅ 오늘의 우주 발견 (카드 뒤집기, 12개 콘텐츠)
- ✅ 우주 퀴즈 (12문제, 랜덤 5문제 출제)
- ✅ 카드 수집 도감 (20장, 4단계 희귀도)
- ✅ 오늘 밤 하늘 (GPS 기반 계절별 천체 안내)
- ✅ 온보딩 (3단계, 닉네임, 시작 카드 선물)

### 게임화 시스템
- XP / 레벨 시스템 (우주 새싹 → 전설의 천문학자)
- 일일 미션 3개 (자정 리셋)
- 6종 일일 퀘스트 풀 (날짜별 순환)
- 10종 업적
- Zustand + AsyncStorage 영구 저장

## 11. AI 기능 설계 (Claude API)

### 모델
`claude-haiku-4-5` (빠른 응답 + 저비용)

### 캐릭터별 시스템 프롬프트 원칙
- 초등학교 4~6학년 눈높이 어휘
- 4문장 이내 응답
- 어려운 용어 즉시 비유로 변환
- 캐릭터 개성 유지 (Steve: 모험가 / Enderman: 신비주의 등)
- 마지막에 항상 탐험/호기심 유발 요소 포함

### 오프라인 폴백
API 불가 시 주요 질문(블랙홀, 외계인, 별 등)에 대한 캐릭터별 로컬 응답 제공
